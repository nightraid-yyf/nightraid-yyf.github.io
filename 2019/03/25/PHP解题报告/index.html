PHP 解题报告

置顶 | Posted on 2019-03-25 | In PHP类

1. PHP LFI

题目要求：调用soulution.php文件
本题是PHP中LFI漏洞的利用，漏洞代码如下:

$filename = ‘pages/‘.(isset($_GET[“file”])?$_GET[“file”]:”welcome”).‘.htm’;

include $filename;

当服务器端get到file的时候，会去调用file.html并显示；那么我们只要将solution.php传入并调用就好。

那么我们尝试输入

?file=solution.php

我们发现返回的是solution.php.html

而我们需要的是solution.php这里就需要用到php的截断符%0，此时输入的代码为

?file=solution.php%%0(%%转义为%)

此时返回的就是solution.php

修改下路径

?file=../../solution.php%%0

得到正确答案

3.Register Globals

本题是利用PHP的register_globals漏洞来进行解题,当设置regiter_globals = on 时会将客户端和代码内部的变量混合在一起

本题的PHP代码就已经设置了register_globals，题目要求我们以admin的账号登陆，查看其PHP代码

从这段代码中可以看到只要我们把login[0]这个变量的值设置为’admin’我们就可以以admin的账号登陆。因为register_globals被设置成on，所以我们只要以get的方式给服务器传递login[0]即可修改对应变量，代码:

?login[0]=admin

4.No Escape

本题是对sql注入的简单利用，目的是让一个人的投票数变为111，首先我们看PHP代码：

这段代码是利用sql语句对 who 这个变量进行赋值，who=getGET(‘vote_for’),并且没有做任何的过滤，那么我们就来进行简单的sql注入

?vote_for=bill`=111--+

便解开此题，注意：这里使用的单引号：` 应与PHP代码中一致

关于sql语句注释的使用:在SQL语句中 -- 代表注释一行但是在我们的get语句中为什么注释是 --+ 呢？

我在网络上CTF大牛的博客上找到了答案：在我们进行sql注入的地方，注释符--后面如果没有空格，就会与单引号或双引号连接形成下面的

--‘

这样的输入，此时sql语句会报错，因为无法识别单引号或双引号的作用，此时+的作用就是代替一个空格使注释变为

-- ’

这样 ‘ 就被正确注释了

6.PHP 0819

我们先看看源码

由源代码看出我们需要让eval=’1337’即可解题

那么先试试

?eval=1337

报错了

我们注意到函数里有这样一行

eval(“\$spaceone = $f”);

eval()的作用是把括号里的字符串当做PHP代码执行，字符串必须满足PHP代码的语法要求，这次我们加上 ; 结尾试试

?eval=1337;

这次没有报错但什么都没有返回

注意我们的最后一行代码

return ($spaceone === ‘1337’);

===的存在使得spaceone必须等于字符串：1337 才能给我们返回正确的结果

首先想到以字符串输入但是代码过滤了’ `两个符号，类型强转(string)也不行，代码第一行’(‘和’)’也被过滤了。

在查找了多种PHP的资料后我学会了一个PHP的技术叫：heredoc

heredoc是PHP引用字符串的一种方式，其用法如下:

$str = <<< s    //需要赋值的变量str，以及一个被 <<< 重定位的标识符 s
strings    //需要输入的字符串strings
s;  //标识符s以及语句结尾;

那么我们的答案就出来了

?eval=<<<s%0a1337%0as;%0a  //%0a为换行符，结尾必须换行否则报错。

此题得解
